QEMU 運行uboot，動態加載內核與文件系統

背景

上一講我們完成了 編譯 QEMU 以及簡單地做了仿真。這一講在 啟動uboot 的基礎上進行，以加強對於 運行地址，加載地址等理解。

有關資料： uboot 與 代碼重定位

有這樣的約定，uboot正常運行的模式稱為自主模式；倒計時被中斷以後的交互模式稱為下載模式。

在Uboot中加載內核與設備樹一般通過網絡(tftp)、讀存儲介質(sd卡\usb等)以及仿真器等方式。

  • uboot通過tftp加載uImage
  • uboot通過sd卡加載uImage

uboot通過tftp加載uImage

為了通過 tftp 加載有關的文件到指定的內存地址，需要先完成下面2個步驟。

在主機搭建tftp服務器

在Linux使用tftp一般以搭建tftp服務器，Windows可以使用軟件完成(例如tftpd32)。
參考文章：《ubuntu 下tftp的安裝、配置、使用》

QEMU網絡功能配置

為了 讓 QEMU 能夠與主機 建立網絡連接，我們採用橋接的網絡連接與Host通信（需要主機內核tun/tap模塊支持）

參考文章：《QEMU 配置網絡》
很可惜我的Ubuntu有問題。折騰了好久網絡都沒有通，但根據以往的開發經驗，教程是可行的。

這裡需要用到 uboot的命令，添加/改動uboot變量有2種方法，一種是在運行的時候通過setenv, saveenv等命令完成的；另外一種就是把有關的命令固化在程序中。
我相信大家對於uboot下載模式下的命令非常熟悉，下面我們簡單介紹在代碼中固化配置的方法。

1）修改uboot代碼並編譯
在include/configs/vexpress_common.h中加入相應的宏定義（相當於步驟4）

// 根據自己的實際情況修改對應的地址
#define CONFIG_IPADDR   192.168.31.240
#define CONFIG_NETMASK  255.255.255.0
#define CONFIG_SERVERIP 192.168.31.241
/* 指定啟動文件為uImage */
#define CONFIG_BOOTFILE "uImage"
/* 指定啟動命令 */
#define CONFIG_BOOTCOMMAND  "tftp 0x60003000 uImage;tftp 0x60500000 vexpress-v2p-ca9.dtb; setenv bootargs'root=/dev/mmcblk0 console=ttyAMA0'; bootm 0x60003000 - 0x60500000"

關於設備樹，可以參考文章: 《設備樹DTS 學習： uboot 傳遞 dtb 給 內核》

實際上，CONFIG_BOOTCOMMAND的值會在uboot啟動以後自動執行，相當於在下載模式輸入下面的指令

tftp 0x60003000 uImage
tftp 0x60500000 vexpress-v2p-ca9.dtb
setenv bootargs'root=/dev/mmcblk0 console=ttyAMA0'
bootm 0x60003000 - 0x60500000

2）指定編譯 kernel 與 dtb

LOADADDR(運行地址) 和uboot的啟動加載位置一致

export ARCH=arm
export CROSS_COMPILE=arm-linux-gnueabi-

make dtbs -j4
make uImage LOADADDR=0x60003000 -j4

uboot通過sd卡加載uImage

uboot通過sd卡加載內核 的步驟和以前差不多。

參考： 《用QEMU模擬運行uboot從SD卡啟動Linux》

製作SD卡鏡像

1）生成一個空的SD卡鏡像

dd if=/dev/zero of=uboot.disk bs=1M count=256
# 注意自己控制 count 大小

2）創建GPT分區，
創建兩個分區（一個用來存放kernel和設備樹，另一個存放根文件系統）

sgdisk -n 0:0:+10M -c 0:kernel uboot.disk
sgdisk -n 0:0:0 -c 0:rootfs uboot.disk

查看分區：

$ sgdisk -p uboot.disk

   1            2048           22527   10.0 MiB    8300  kernel

3）尋找一個空閒的loop設備 並 將SD卡鏡像映射到loop設備上

LOOPDEV=`losetup -f` 
sudo losetup $LOOPDEV  uboot.disk
sudo partprobe $LOOPDEV

此時會看到/dev/loop0p1 和/dev/loop0p2 兩個節點

$ ls /dev/loop*
/dev/loop0    /dev/loop1  /dev/loop4  /dev/loop7
/dev/loop0p1  /dev/loop2  /dev/loop5  /dev/loop-control
/dev/loop0p2  /dev/loop3  /dev/loop6

4）格式化

sudo mkfs.ext4 /dev/loop0p1
sudo mkfs.ext4 /dev/loop0p2

5）掛載

mkdir p1 p2
sudo mount -t ext4 /dev/loop0p1 p1
sudo mount -t ext4 /dev/loop0p2 p2

6）拷貝文件

# 將 zImage 和 dtb 拷貝到 p1
# schips @ ubuntu in ~/arm/sc/linux-4.14.14 [17:55:39]
$ sudo cp arch/arm/boot/zImage p1
$ sudo cp arch/arm/boot/dts/vexpress-v2p-ca9.dtb p1

# 將 文件系統中的文件拷貝到 p2
## 因為在上一講我已經做好了一個 ext3的鏡像，所以我直接使用了

# schips @ ubuntu in ~/arm/sc/linux-4.14.14 [17:59:24] C:1
$ sudo mount -t ext3 ../busybox-1.27.0/a9rootfs.ext3 _tmp -o loop
$ sudo cp _tmp/* p2 -arf

7）取消掛載

sudo umount p1 p2
sudo losetup -d /dev/loop0

加載kernel、設備樹

加載sd卡分區中指定的文件（內核）到指定的內存： load mmc 0:1 0x60008000 zImage或者ext4load mmc 0:1 0x60008000 zImage

=> load mmc 0:1 0x60008000 zImage
3961960 bytes read in 11446 ms (337.9 KiB/s)

加載sd卡分區中指定的文件（設備樹）到指定的內存：load mmc 0:1 0x61000000 vexpress-v2p-ca9.dtb 或者 ext4load mmc 0:1 0x61000000 vexpress-v2p-ca9.dtb

=> load mmc 0:1 0x61000000 vexpress-v2p-ca9.dtb
14692 bytes read in 35 ms (409.2 KiB/s)

設置bootargs

setenv bootargs 'root=/dev/mmcblk0p2 rw rootfstype=ext4 rootwait earlycon console=tty0 console=ttyAMA0 init=/linuxrc ignore_loglevel'

引導內核

=> bootz 0x60008000 - 0x61000000

關於 uboot 啟動命令可以參考：《Uboot 命令行 介紹》

固化啟動命令

如果沒有問題的話，那麼可以在uboot的代碼中將上面的命令固化進程序中
在include/configs/vexpress_common.h中修改CONFIG_BOOTCOMMAND
改成

#define CONFIG_BOOTCOMMAND  "load mmc 0:1 0x60008000 zImage ;load mmc 0:1 0x61000000 vexpress-v2p-ca9.dtb; setenv bootargs \"root=/dev/mmcblk0p2 earlycon console=ttyAMA0\"; bootz 0x60008000 - 0x61000000"

再重新編譯即可。
