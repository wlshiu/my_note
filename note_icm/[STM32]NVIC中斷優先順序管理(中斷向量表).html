【STM32】NVIC中斷優先順序管理（中斷向量表）

其他 ‧ 發表 2019-02-01

STM32F1xx官方資料：
《STM32中文參考手冊V10》-第9章 中斷和事件

Cortex-M3核心支援256箇中斷，其中包含了16個核心中斷（異常）和240個外部中斷，並且具有256級的可程式設計中斷設定。但是，STM32並沒有使用CM3核心的全部東西，而是隻用了它的一部分。STM32有84箇中斷，包括16個核心中斷（異常）和68個可遮蔽中斷，具有16級可程式設計的中斷優先順序。而STM32F103系列上面，16個核心中斷（異常）不變，而可遮蔽中斷只有60個（在107系列才有68個）。

注意一下：CM3的外部中斷和STM32的外部中斷不是一個概念。CM3：除了核心異常之外的都是外部中斷；STM32：外部中斷EXTI只有6個

其實，核心中斷也叫核心異常。

  • 中斷是指系統停止當前正在執行的程式轉而其他服務，可能是程式接收了比自身高優先順序的請求，或者是人為設定中斷，中斷是屬於正常現象。 
  • 異常是指由於cpu本身故障、程式故障或者請求服務等引起的錯誤，異常屬於不正常現象。

因此，下面就直接介紹一下STM32F103系列的16個核心中斷（異常）、60箇中斷：



在中斷向量表中從優先順序7-66（中斷號從0-59）代表著STM32F103的60箇中斷。優先順序號越小，優先順序越高。當表中的某處異常或中斷被觸發，程式計數器指標（PC）將跳轉到該異常或中斷的地址處執行，該地址處存放這一條跳轉指令，跳轉到該異常或中斷的服務函式處執行相應的功能。因此，異常和中斷向量表只能用彙編語言編寫。

在MDK中，有標準的異常和中斷向量表文件可以使用（startup_stm32f10x_hd.s），在其中標明瞭中斷處理函式的名稱，不能隨意定義。而中斷通道NVIC_IRQChannel（即IRQn_Type型別）是在stm32f10x.h檔案中進行了巨集定義。

什麼是NVIC？即巢狀向量中斷控制器(Nested Vectored Interrupt Controller)。CM3的中有一個強大而方便的NVIC，它是屬於Cortex核心的器件，中斷向量表中60箇中斷都由它來處理。NVIC是Cortex-M3核心的一部分，關於它的資料不在《STM32的技術參考手冊》中，應查閱ARM公司的《Cortex-M3技術參考手冊》。Cortex-M3的向量中斷統一由NVIC管理。

NVIC的核心功能是中斷優先順序分組、中斷優先順序的配置、讀中斷請求標誌、清除中斷請求標誌、使能中斷、清除中斷等，它控制著STM32中斷向量表中中斷號為0-59的60箇中斷！！外部中斷訊號從核外發出，訊號最終要傳遞到NVIC(巢狀向量中斷控制器)。NVIC跟核心緊密耦合，它控制著整個晶片中斷的相關功能。

STM32中斷優先順序分組

中斷優先順序分組暫存器

這60箇中斷，怎麼管理呢？這就涉及到STM32的中斷分組。STM32可以將中斷分成5個組，分別為組0-4；同時，對每個中斷設定一個搶佔優先順序和響應優先順序。分組配置是由SCB->AIRCR暫存器的bit10-8來定義的。SCB->AIRCR是在哪裡的呢？由於這是CM3核心定義的，在文件《Cortex-M3權威指南(中文)》中能查詢到。

具體的分配關係如下所示：

AIRCR中斷分組設定表
組AIRCR[10:8]IP bit[7:4]分配情況分配結果
01110:40位搶佔優先順序，4位響應優先順序
11101:31位搶佔優先順序，3位響應優先順序
21012:22位搶佔優先順序，2位響應優先順序
31003:13位搶佔優先順序，1位響應優先順序
40114:04位搶佔優先順序，0位響應優先順序

其中AIRCR暫存器來確定是用哪種分組，IP暫存器是相對應於那種分組搶佔優先順序和響應優先順序的分配比例。例如組設定成3，那麼此時所有的60箇中斷優先暫存器高4位中的最高3位是搶佔優先順序，低1位為響應優先順序。CM3中定義了8個Bit用於設定中斷源的優先順序，而STM32只選用其中的4個Bit。

搶佔優先順序的級別高於響應優先順序，而數值越小所代表的的優先順序越高。 

介紹一下搶佔優先順序、響應優先順序的區別：

  1. 高優先順序的搶佔優先順序是可以打斷正在進行的低搶佔優先順序中斷的；
  2. 搶佔優先順序相同的中斷，高響應優先順序不可以打斷低響應優先順序的中斷；
  3. 搶佔優先順序相同的中斷，當兩個中斷同時發生的情況下，哪個響應優先順序高，哪個先執行；
  4. 如果兩個中斷的搶佔優先順序和響應優先順序都是一樣的話，則看哪個中斷先發生就先執行；

除此之外有兩點需要注意：

  1. 打斷的情況只會與搶佔優先順序有關， 和響應優先順序無關！
  2. 一般情況下，系統程式碼執行過程中，只設置一次中斷優先順序分組，比如分組2，設定好分組之後一般不會再改變分組。隨意改變分組會導致中斷管理混亂，程式出現意想不到的執行結果。

中斷優先順序分組庫函式

CM3核的優先順序分組方式，使用的設定函式NVIC_SetPriorityGrouping()。

接下來介紹STM32的中斷優先順序分組函式NVIC_PriorityGroupConfig()，用來進行中斷分組設定的，此函式是在韌體庫下misc.c檔案中（檔案目錄是：

STM32F10x_StdPeriph_Lib_V3.5.0\Libraries\STM32F10x_StdPeriph_Driver\src\misc.c）：

void NVIC_PriorityGroupConfig(uint32_t NVIC_PriorityGroup)
{
  /* Check the parameters */
  assert_param(IS_NVIC_PRIORITY_GROUP(NVIC_PriorityGroup));
  
  /* Set the PRIGROUP[10:8] bits according to NVIC_PriorityGroup value */
  SCB->AIRCR = AIRCR_VECTKEY_MASK | NVIC_PriorityGroup;
}

函式的引數的取值，是在同文件中進行巨集定義的：

#define NVIC_PriorityGroup_0         ((uint32_t)0x700) /*!< 0 bits for pre-emption priority
                                                            4 bits for subpriority */
#define NVIC_PriorityGroup_1         ((uint32_t)0x600) /*!< 1 bits for pre-emption priority
                                                            3 bits for subpriority */
#define NVIC_PriorityGroup_2         ((uint32_t)0x500) /*!< 2 bits for pre-emption priority
                                                            2 bits for subpriority */
#define NVIC_PriorityGroup_3         ((uint32_t)0x400) /*!< 3 bits for pre-emption priority
                                                            1 bits for subpriority */
#define NVIC_PriorityGroup_4         ((uint32_t)0x300) /*!< 4 bits for pre-emption priority
                                                            0 bits for subpriority */

STM32中斷優先順序管理

中斷優先順序設定暫存器

分組設定好了之後，怎麼設定單箇中斷的搶佔優先順序和響應優先順序？

MDK為與NVIC相關的暫存器定義瞭如下的結構體，控制著中斷向量表中60箇中斷（由於與中斷核心有關，定義在core_cm3.h檔案中）：

typedef struct
{
  __IO uint32_t ISER[8];                      /*!< Offset: 0x000  Interrupt Set Enable Register           */
       uint32_t RESERVED0[24];                                   
  __IO uint32_t ICER[8];                      /*!< Offset: 0x080  Interrupt Clear Enable Register         */
       uint32_t RSERVED1[24];                                    
  __IO uint32_t ISPR[8];                      /*!< Offset: 0x100  Interrupt Set Pending Register          */
       uint32_t RESERVED2[24];                                   
  __IO uint32_t ICPR[8];                      /*!< Offset: 0x180  Interrupt Clear Pending Register        */
       uint32_t RESERVED3[24];                                   
  __IO uint32_t IABR[8];                      /*!< Offset: 0x200  Interrupt Active bit Register           */
       uint32_t RESERVED4[56];                                   
  __IO uint8_t  IP[240];                      /*!< Offset: 0x300  Interrupt Priority Register (8Bit wide) */
       uint32_t RESERVED5[644];                                  
  __O  uint32_t STIR;                         /*!< Offset: 0xE00  Software Trigger Interrupt Register     */
}  NVIC_Type;

我們依次介紹一下這些暫存器：

先介紹幾個暫存器組長度為8，這些暫存器是32位暫存器。由於STM32只有60個可遮蔽中斷，8個32位暫存器中只需要2個就有64位了，每1位控制一箇中斷。

  • ISER[8]（Interrupt Set-Enable Registers）：中斷使能暫存器。其中只使用到了ISER[0]和ISER[1]，ISER[0]的bit0~bit31分別對應中斷0~31。ISER[1]的bit0~27對應中斷32~59。要使能某個中斷，就必須設定相應的ISER位為1，使該中斷被使能（這僅僅是使能，還要配合中斷分組、遮蔽、I/O口對映等設定才算完整）。具體每一位對應哪個中斷參考stm32f103x.h裡面第140行。
  • ICER[8]（Interrupt Clear-Enable Registers）：中斷移除暫存器。該暫存器的作用於ISER相反。這裡專門設定一個ICER來清除中斷位，而不是向ISER位寫0，是因為NVIC的暫存器寫1有效，寫0無效。
  • ISPR[8]（Interrupt Set-Pending Registers）：中斷掛起控制暫存器。通過置1可以將正在進行的中斷掛起，執行同級或者更高級別的中斷。寫0無效。
  • ICPR[8]（Interrupt Clear-Pending Registers）：中斷解掛控制暫存器。通過置1可以將正在掛起的中斷解掛。寫0無效。
  • IABR[8]（Interrupt Active-Bit Registers）：中斷啟用標誌位暫存器。這是一個只讀暫存器，可以知道當前在執行的中斷是哪一個（為1），在中斷執行完後硬體自動清零。

最後，介紹一個暫存器組長度為240，這個暫存器為8位暫存器。240個8位暫存器，每個中斷使用一個暫存器來確定優先順序。由於CM3由240個外部中斷，所以這個暫存器組的數目就是240（注意與上面暫存器的區別，一個是一個暫存器控制一個，一個是一位控制一個）。

  • IP[240]（Interrupt Priority Registers）：中斷優先順序控制的暫存器。這是用來控制每個中斷的優先順序。由於STM32F10x系列一共60個可遮蔽中斷，故使用IP[59]~IP[0]。其中每個IP暫存器的高4位[7:4]用來設定搶佔和響應優先順序（根據分組），低4位沒有用到。而兩個優先順序各佔幾個位又要由上面講到的中斷優先順序分組決定。

中斷優先順序設定庫函式

接下來介紹如何使用庫函式實現中斷優先順序管理，這裡使用NVIC_Init()函式來進行對每個中斷優先順序的設定（misc.c檔案中）：

void NVIC_Init(NVIC_InitTypeDef* NVIC_InitStruct)
{
  uint32_t tmppriority = 0x00, tmppre = 0x00, tmpsub = 0x0F;
  
  /* Check the parameters */
  assert_param(IS_FUNCTIONAL_STATE(NVIC_InitStruct->NVIC_IRQChannelCmd));
  assert_param(IS_NVIC_PREEMPTION_PRIORITY(NVIC_InitStruct->NVIC_IRQChannelPreemptionPriority));  
  assert_param(IS_NVIC_SUB_PRIORITY(NVIC_InitStruct->NVIC_IRQChannelSubPriority));
    
  if (NVIC_InitStruct->NVIC_IRQChannelCmd != DISABLE)
  {
    /* Compute the Corresponding IRQ Priority --------------------------------*/    
    tmppriority = (0x700 - ((SCB->AIRCR) & (uint32_t)0x700))>> 0x08;
    tmppre = (0x4 - tmppriority);
    tmpsub = tmpsub >> tmppriority;

    tmppriority = (uint32_t)NVIC_InitStruct->NVIC_IRQChannelPreemptionPriority << tmppre;
    tmppriority |=  NVIC_InitStruct->NVIC_IRQChannelSubPriority & tmpsub;
    tmppriority = tmppriority << 0x04;
        
    NVIC->IP[NVIC_InitStruct->NVIC_IRQChannel] = tmppriority;
    
    /* Enable the Selected IRQ Channels --------------------------------------*/
    NVIC->ISER[NVIC_InitStruct->NVIC_IRQChannel >> 0x05] =
      (uint32_t)0x01 << (NVIC_InitStruct->NVIC_IRQChannel & (uint8_t)0x1F);
  }
  else
  {
    /* Disable the Selected IRQ Channels -------------------------------------*/
    NVIC->ICER[NVIC_InitStruct->NVIC_IRQChannel >> 0x05] =
      (uint32_t)0x01 << (NVIC_InitStruct->NVIC_IRQChannel & (uint8_t)0x1F);
  }
}

其中，NVIC_InitTypeDef為一個結構體，它的成員變數為：

typedef struct
{
  uint8_t NVIC_IRQChannel;                    /*!< Specifies the IRQ channel to be enabled or disabled.
                                                   This parameter can be a value of @ref IRQn_Type 
                                                   (For the complete STM32 Devices IRQ Channels list, please
                                                    refer to stm32f10x.h file) */

  uint8_t NVIC_IRQChannelPreemptionPriority;  /*!< Specifies the pre-emption priority for the IRQ channel
                                                   specified in NVIC_IRQChannel. This parameter can be a value
                                                   between 0 and 15 as described in the table @ref NVIC_Priority_Table */

  uint8_t NVIC_IRQChannelSubPriority;         /*!< Specifies the subpriority level for the IRQ channel specified
                                                   in NVIC_IRQChannel. This parameter can be a value
                                                   between 0 and 15 as described in the table @ref NVIC_Priority_Table */

  FunctionalState NVIC_IRQChannelCmd;         /*!< Specifies whether the IRQ channel defined in NVIC_IRQChannel
                                                   will be enabled or disabled. 
                                                   This parameter can be set either to ENABLE or DISABLE */   
} NVIC_InitTypeDef;

NVIC_InitTypeDef結構體有4個成員變數：

  1. NVIC_IRQChannel：定義初始化的是哪一個中斷，這個可以在stm32f10x.h檔案中查到每個中斷對應的名字，如USART1_IRQn；
  2. NVIC_IRQChannelPreemptionPriority：定義此中斷的搶佔優先級別；
  3. NVIC_IRQChannelSubPriority：定義此中斷的響應優先級別；
  4. NVIC_IRQChannelCmd：該中斷是否使能。

其實我們看NVIC_Init()函式內部使能中斷，也是通過ISER暫存器配置的。這與我麼之前的內容並不矛盾。函式內部使用NVIC->ISER，而NVIC是一個巨集定義，

#define NVIC     ((NVIC_Type *) NVIC_BASE)     /*!< NVIC configuration struct         */
也就是直接操作結構體來實現操作ISER暫存器。具體原理見【STM32】MDK中暫存器地址名稱對映分析。

比如，使能串列埠1中斷，搶佔優先順序為1，響應優先順序為2，初始化的方法為：

NVIC_InitTypeDef   NVIC_InitStructure;
NVIC_InitStructure.NVIC_IRQChannel = USART1_IRQn;//串列埠1中斷
NVIC_InitStructure.NVIC_IRQChannelPreemptionPriority=1 ;// 搶佔優先順序為1
NVIC_InitStructure.NVIC_IRQChannelSubPriority = 2;// 子優先順序位2
NVIC_InitStructure.NVIC_IRQChannelCmd = ENABLE;//IRQ通道使能
NVIC_Init(&NVIC_InitStructure);	//根據上面指定的引數初始化NVIC暫存器

總結與分析

最後總結一下中斷優先順序設定的步驟：

1、系統執行後先設定中斷優先順序分組。呼叫函式：

void NVIC_PriorityGroupConfig(uint32_t NVIC_PriorityGroup);

整個系統執行過程中，只設置一次中斷分組；

2、針對每個中斷，設定對應的搶佔優先順序和響應優先順序：

void NVIC_Init(NVIC_InitTypeDef* NVIC_InitStruct);

3、如果需要掛起/解掛，檢視中斷當前啟用狀態，分別呼叫相關函式即可。
