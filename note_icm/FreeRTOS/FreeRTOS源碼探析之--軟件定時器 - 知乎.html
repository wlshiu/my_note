軟件定時器是FreeRTOS中的一個重要模塊，使用軟件定時器可以方便的實現一些與超時或週期性相關的功能，本篇從FreeRTOS的源碼入手，來分析FreeRTOS軟件定時器的運行機理。

1 基礎知識

1.1 軟件定時器與硬件定時器的區別

硬件定時器

  • 每次在定時時間到達之後就會自動觸發一個中斷，用戶在中斷服務函數中處理信息
  • 硬件定時器的精度一般很高，可以達到納秒級別
  • 硬件定時器是芯片本身提供的定時功能

軟件定時器

  • 指定時間到達後要調用回調函數（也稱超時函數），用戶在回調函數中處理信息
  • 硬件定時器的定時精度與系統時鐘的週期有關，一般系統利用SysTick作為軟件定時器的基礎時鐘，系統節拍配置為FreeRTOSConfig.h中的configTICK_RATE_HZ，默認是1000，那麼系統的時鐘節拍週期就為1ms
  • 軟件定時器是由操作系統提供的一類系統接口

注意：軟件定時器回調函數的上下文是任務，回調函數要快進快出，且回調函數中不能有任何阻塞任務運行的情況，如vTaskDelay()以及其它能阻塞任務運行的函數。

1.2 軟件定時器的兩種工作模式

FreeRTOS提供的軟件定時器支持單次模式和週期模式

  • 單次模式：當用戶創建了定時器並啟動了定時器後，定時時間到了，只執行一次回調函數之後就將該定時器刪除，不再重新執行。
  • 週期模式：這個定時器會按照設置的定時時間循環執行回調函數，直到用戶將定時器刪除

2 軟件定時器工作原理

通過查看FreeRTOS的源碼，可以發現，軟件定時器的運行原理實際是FreeRTOS 通過一個 prvTimerTask任務（也叫守護任務Daemon）管理軟定時器，它是在啟動調度器時自動創建的。另外，軟件定時器在FreeRTOS中是可選功能，如果需要使用軟件定時器，需要設置 FreeRTOSConfig.h 中的宏定義configUSE_TIMERS為1 。

2.3 創建列表與消息隊列

由於系統節拍採用32位變量進行計數，總有一天會溢出，所以軟件定時器使用了兩個列表：

  • 當前定時器列表 pxCurrentTimerList ：系統新創建並激活的定時器都會以超時時間升序的方式插入到pxCurrentTimerList列表中。系統在定時器任務中掃瞄pxCurrentTimerList中的第一個定時器，看是否已超時，若已經超時了則調用軟件定時器回調函數，否則將定時器任務掛起。
  • 溢出定時器列表pxOverflowTimerList：在軟件定時器溢出的時候使用，作用與pxCurrentTimerList一致。

定時器列表會按照喚醒時間從早到晚掛接在當前定時器列表中，喚醒時間如果溢出了就掛接在溢出定時器列表中。當系統節拍溢出之後，兩個列表的功能會進行交換，即當前列表變為溢出列表，溢出列表變為當前列表。

此外，FreeRTOS的軟件定時器還使用了一個消息隊列xTimerQueue，利用「定時器命令隊列」向軟件定時器任務發送一些命令，任務在接收到命令就會去處理命令對應的程序，比如啟動定時器，停止定時器，復位、刪除、改變週期等。

假如定時器任務處於阻塞狀態，我們又需要馬上再添加一個軟件定時器的話，就是採用這種消息隊列命令的方式進行添加，才能喚醒處於等待狀態的定時器任務，並且在任務中將新添加的軟件定時器添加到軟件定時器列表中

（註：事件標誌組在中斷中設置事件標誌，實際也是通過隊列發送消息給軟件定時器任務來執行）

2.5 軟件定時器打包命令與發送

該函數將命令打包成隊列項發送給xTimerQueue消息隊列，由軟件定時器任務（守護任務來）接收並進行處理。 

/* 軟件定時器打包命令與發送 */
BaseType_t xTimerGenericCommand( TimerHandle_t xTimer, const BaseType_t xCommandID, const TickType_t xOptionalValue, BaseType_t * const pxHigherPriorityTaskWoken, const TickType_t xTicksToWait )
{
    BaseType_t xReturn = pdFAIL;
    DaemonTaskMessage_t xMessage;

    configASSERT( xTimer );

    if( xTimerQueue != NULL )
    {
        /* 命令碼 */
        xMessage.xMessageID = xCommandID;
        /* 命令有效值 */
        xMessage.u.xTimerParameters.xMessageValue = xOptionalValue;
        /* 定時器句柄 */
        xMessage.u.xTimerParameters.pxTimer = xTimer;

        /* 不帶中斷命令 */
        if(xCommandID < tmrFIRST_FROM_ISR_COMMAND)
        {
            /* 調度器正在運行 */
            if(xTaskGetSchedulerState() == taskSCHEDULER_RUNNING)
            {
                /* 將命令消息發送到隊列，可以阻塞一定時間 */
                xReturn = xQueueSendToBack(xTimerQueue, &xMessage, xTicksToWait);
            }
            /* 調度器不在運行 */
            else
            {
                /* 將命令消息發送到隊列 ，不帶阻塞時間*/
                xReturn = xQueueSendToBack(xTimerQueue, &xMessage, tmrNO_DELAY);
            }
        }
        /* 帶中斷命令 */
        else
        {
            /* 將命令消息發送到隊列 */
            xReturn = xQueueSendToBackFromISR(xTimerQueue, &xMessage, pxHigherPriorityTaskWoken);
        }

        traceTIMER_COMMAND_SEND( xTimer, xCommandID, xOptionalValue, xReturn );
    }
    else
    {
        mtCOVERAGE_TEST_MARKER();
    }

    return xReturn;
}

2.6 軟件定時器任務基本功能（三部分）

軟件定時器任務的具體內容可分為三部分：

  • 獲取最近一次定時器超時時間
  • 處理超時的定時器或者讓隊列阻塞
  • 處理隊列接收到的命令

三部分不斷循環處理實現Daemon任務。

    #if( configUSE_DAEMON_TASK_STARTUP_HOOK == 1 )
    ...略去部分代碼
    #endif /* configUSE_DAEMON_TASK_STARTUP_HOOK */

    {

3 軟件定時器任務三部分功能分析

先來一張整體結構圖：

首先是從定時器列表中獲取下一次的溢出時間，因為各定時器的溢出時間是按照升序排列的，因此只需獲取下一次的溢出時間。

3.1 獲取下一個定時超時時間

/* 獲取下一次的定時器超時時間 */
static TickType_t prvGetNextExpireTime( BaseType_t * const pxListWasEmpty )
{
TickType_t xNextExpireTime;

    /* 判斷當前定時器列表是否為空 */
    *pxListWasEmpty = listLIST_IS_EMPTY( pxCurrentTimerList );

    /* 當前列表非空 */
    if( *pxListWasEmpty == pdFALSE )
    {
        /* 獲取最近超時時間 */
        xNextExpireTime = listGET_ITEM_VALUE_OF_HEAD_ENTRY( pxCurrentTimerList );
    }
    else /* 當前列表為空 */
    {
        /*超時時間設為0，使任務非阻塞 */
        xNextExpireTime = ( TickType_t ) 0U;
    }

    return xNextExpireTime;
}

3.2 處理或阻塞軟件定時器任務

那系統如何處理軟件定時器列表？系統在不斷運行，而xTimeNow（xTickCount）隨著SysTick的觸發一直在增長，在軟件定時器任務運行的時候會獲取下一個要喚醒的定時器:

  • 比較當前系統時間xTimeNow是否大於或等於下一個定時器喚醒時間xTicksToWait
  • 若大於則表示已經超時，定時器任務將會調用對應定時器的回調函數
  • 否則將軟件定時器任務掛起，直至下一個要喚醒的軟件定時器時間到來或者接收到命令消息
/* 處理或阻塞軟件定時器任務 */
static void prvProcessTimerOrBlockTask( const TickType_t xNextExpireTime, BaseType_t xListWasEmpty )
{
    TickType_t xTimeNow;
    BaseType_t xTimerListsWereSwitched;

    /* 掛起調度器 */
    vTaskSuspendAll();
    {
        /* 獲取當前時間，並判斷是否需要切換定時器列表，如果需要則切換 */
        xTimeNow = prvSampleTimeNow( &xTimerListsWereSwitched );

        /* 定時器列表沒有切換 */
        if( xTimerListsWereSwitched == pdFALSE )
        {
            /* 當前列表中有定時器，且下次喚醒時間小於當前時間，即超時了 */
            if( ( xListWasEmpty == pdFALSE ) && ( xNextExpireTime <= xTimeNow ) )
            {
                /* 解除調度器掛起 */
                ( void )xTaskResumeAll();
                /* 處理超時的定時器 */
                prvProcessExpiredTimer( xNextExpireTime, xTimeNow );
            }
            else/* 定時器列表為空，或者沒有超時 */
            {
                /* 定時器列表為空 */
                if( xListWasEmpty != pdFALSE )
                {
                    /* 判斷溢出列表是否為空，如果兩個列表都為空，則無限期阻塞 */
                    xListWasEmpty = listLIST_IS_EMPTY( pxOverflowTimerList );
                }

                /* 定時器定時時間還沒到，將當前任務掛起，讓隊列按照給定的時間進行阻塞 */
                vQueueWaitForMessageRestricted( xTimerQueue, ( xNextExpireTime - xTimeNow ), xListWasEmpty );
                /* 解除調度器掛起 */
                if( xTaskResumeAll() == pdFALSE )
                {
                    /* 申請切換任務 */
                    portYIELD_WITHIN_API();
                }
                else
                {
                    mtCOVERAGE_TEST_MARKER();
                }
            }
        }
        else
        {
            /* 解除調度器掛起 */
            ( void ) xTaskResumeAll();
        }
    }
}

3.2.1 獲取當前時間並決定是否切換列表

static TickType_t prvSampleTimeNow( BaseType_t * const pxTimerListsWereSwitched )
{
    TickType_t xTimeNow;
    /*靜態變量 記錄上一次調用時系統節拍值*/
    PRIVILEGED_DATA static TickType_t xLastTime = ( TickType_t ) 0U; 
    /*獲取本次調用節拍結束器值*/
    xTimeNow = xTaskGetTickCount();

    /*判斷節拍計數器是否溢出過*/
    if( xTimeNow < xLastTime )
    {
        /*發生溢出，處理當前鏈表上所有定時器並切換管理鏈表*/
        prvSwitchTimerLists();
        *pxTimerListsWereSwitched = pdTRUE;
    }
    else
    {
        *pxTimerListsWereSwitched = pdFALSE;
    }
    /*更新時間記錄*/
    xLastTime = xTimeNow;

    return xTimeNow;
}

可以看到， 該函數每次調用都會記錄當前系統節拍時間（TickCount）， 下一次調用，通過比較相鄰兩次調用的值判斷節拍計數器是否溢出。當系統節拍計數器溢出， 必須切換計時器列表。如果當前計時器列表中仍然引用任何計時器，那麼它們一定已經過期，應該在切換列表之前進行處理。

切換列表的具體內容如下：

static void prvSwitchTimerLists( void )
{
    TickType_t xNextExpireTime, xReloadTime;
    List_t *pxTemp;
    Timer_t *pxTimer;
    BaseType_t xResult;

    /* 列表非空，循環處理，直至將該列表處理完 */
    while( listLIST_IS_EMPTY( pxCurrentTimerList ) == pdFALSE )
    {
        xNextExpireTime = listGET_ITEM_VALUE_OF_HEAD_ENTRY( pxCurrentTimerList );

        /* 從列表中移除軟件定時器 */
        pxTimer = ( Timer_t * ) listGET_OWNER_OF_HEAD_ENTRY( pxCurrentTimerList );
        ( void ) uxListRemove( &( pxTimer->xTimerListItem ) );
        traceTIMER_EXPIRED( pxTimer );

        /*執行回調函數*/
        pxTimer->pxCallbackFunction( ( TimerHandle_t ) pxTimer );

        /*對於週期定時器*/
        if( pxTimer->uxAutoReload == ( UBaseType_t ) pdTRUE )
        {
            /*計算重新加載值：下個溢出時間 + 定時週期*/
            xReloadTime = ( xNextExpireTime + pxTimer->xTimerPeriodInTicks );
            /*如果重新加載值>下個溢出時間，應該將計時器重新插入當前列表，以便在此循環中再次處理它*/
            if( xReloadTime > xNextExpireTime )
            {
                listSET_LIST_ITEM_VALUE( &( pxTimer->xTimerListItem ), xReloadTime );
                listSET_LIST_ITEM_OWNER( &( pxTimer->xTimerListItem ), pxTimer );
                vListInsert( pxCurrentTimerList, &( pxTimer->xTimerListItem ) );
            }
            else/*否則，應該發送一個命令來重新啟動計時器，以確保它只插入到列表之後列表已被交換*/
            {
                xResult = xTimerGenericCommand( pxTimer, tmrCOMMAND_START_DONT_TRACE, xNextExpireTime, NULL, tmrNO_DELAY );
                configASSERT( xResult );
                ( void ) xResult;
            }
        }
        else
        {
            mtCOVERAGE_TEST_MARKER();
        }
    }

    pxTemp = pxCurrentTimerList;
    pxCurrentTimerList = pxOverflowTimerList;
    pxOverflowTimerList = pxTemp;
}

3.2.2 處理超時的定時器

/* 處理超時的定時器 */
static void prvProcessExpiredTimer( const TickType_t xNextExpireTime, const TickType_t xTimeNow )
{
    BaseType_t xResult;
    /* 獲取最近的超時定時器 */
    Timer_t *const pxTimer = ( Timer_t * )listGET_OWNER_OF_HEAD_ENTRY( pxCurrentTimerList );

    /* 將最近的超時定時器從活躍列表中移除 */
    (void)uxListRemove( &( pxTimer->xTimerListItem ) );
    traceTIMER_EXPIRED(pxTimer);

    /* 週期定時 */
    if( pxTimer->uxAutoReload == ( UBaseType_t )pdTRUE )
    {
        /* 重新計算超時時間並加入活躍列表，如果下一次超時時間都已經過了 */
        if( prvInsertTimerInActiveList( pxTimer, ( xNextExpireTime + pxTimer->xTimerPeriodInTicks ), xTimeNow, xNextExpireTime ) != pdFALSE )
        {
            /* 通知守護任務來處理（將定時器插入活躍列表） */
            xResult = xTimerGenericCommand( pxTimer, tmrCOMMAND_START_DONT_TRACE, xNextExpireTime, NULL, tmrNO_DELAY );
            configASSERT( xResult );
            ( void )xResult;
        }
        else
        {
            mtCOVERAGE_TEST_MARKER();
        }
    }
    else
    {
        mtCOVERAGE_TEST_MARKER();
    }

    /* 調用回調函數 */
    pxTimer->pxCallbackFunction( ( TimerHandle_t )pxTimer );
}

3.2.3 讓隊列按照給定的時間進行阻塞

回顧prvProcessTimerOrBlockTask()函數，定時器定時時間還沒到，將當前任務掛起，直到定時器到期才喚醒或者收到命令的時候喚醒：

/* 讓隊列按照給定的時間進行阻塞 */
void vQueueWaitForMessageRestricted( QueueHandle_t xQueue, TickType_t xTicksToWait, const BaseType_t xWaitIndefinitely )
{
    Queue_t *const pxQueue = xQueue;

    /* 鎖定隊列 */
    prvLockQueue( pxQueue );

    /* 隊列為空 */
    if( pxQueue->uxMessagesWaiting == ( UBaseType_t )0U )
    {
        /* 將任務插入等待接收隊列項而阻塞的事件列表，並加入延時列表進行阻塞延時 */
        vTaskPlaceOnEventListRestricted( &( pxQueue->xTasksWaitingToReceive ), xTicksToWait, xWaitIndefinitely );
    }
    /* 隊列不為空 */
    else
    {
        mtCOVERAGE_TEST_MARKER();
    }
    /* 解鎖隊列 */
    prvUnlockQueue(pxQueue);
}

3.3 處理命令隊列中接收的消息

用戶將需要處理的定時器命令發送到定時器的消息隊列， Daemon 任務每次執行期間回去讀取並執行，下面看看該函數的具體內容：

/*處理命令隊列中接收的消息*/
static void prvProcessReceivedCommands( void )
{
    DaemonTaskMessage_t xMessage;
    Timer_t *pxTimer;
    BaseType_t xTimerListsWereSwitched, xResult;
    TickType_t xTimeNow;

    /*消息隊列接收*/
    while( xQueueReceive( xTimerQueue, &xMessage, tmrNO_DELAY ) != pdFAIL )
    {
        #if ( INCLUDE_xTimerPendFunctionCall == 1 )
        {
            /* 命令碼小於等於0 (事件標誌組中斷中置位的命令)*/
            if( xMessage.xMessageID < ( BaseType_t ) 0 )
            {
                const CallbackParameters_t * const pxCallback = &( xMessage.u.xCallbackParameters );

                configASSERT( pxCallback );

                /* 執行回調函數 */
                pxCallback->pxCallbackFunction( pxCallback->pvParameter1, pxCallback->ulParameter2 );
            }
            else
            {
                mtCOVERAGE_TEST_MARKER();
            }
        }
        #endif /* INCLUDE_xTimerPendFunctionCall */

        /* 命令碼大於等於0 (軟件定時器命令)*/
        if( xMessage.xMessageID >= ( BaseType_t ) 0 )
        {
            /* 定時器句柄 */
            pxTimer = xMessage.u.xTimerParameters.pxTimer;

            /* 定時器隊列項包含該定時器 */
            if( listIS_CONTAINED_WITHIN( NULL, &( pxTimer->xTimerListItem ) ) == pdFALSE )
            {
                /* 移除該定時器 */
                ( void ) uxListRemove( &( pxTimer->xTimerListItem ) );
            }
            else
            {
                mtCOVERAGE_TEST_MARKER();
            }

            traceTIMER_COMMAND_RECEIVED( pxTimer, xMessage.xMessageID, xMessage.u.xTimerParameters.xMessageValue );

            /* 獲取當前時間，並判斷是否需要切換定時器列表，如果需要則切換 */
            xTimeNow = prvSampleTimeNow( &xTimerListsWereSwitched );

            /* 消息類型 */
            switch( xMessage.xMessageID )
            {
                /* 定時器啟動或者復位 */
                case tmrCOMMAND_START :
                case tmrCOMMAND_START_FROM_ISR :
                case tmrCOMMAND_RESET :
                case tmrCOMMAND_RESET_FROM_ISR :
                case tmrCOMMAND_START_DONT_TRACE :
                    /* 計算超時時間，超時時間沒過加入活躍列表，超時時間已過返回pdTrue */
                    if( prvInsertTimerInActiveList( pxTimer,  xMessage.u.xTimerParameters.xMessageValue + pxTimer->xTimerPeriodInTicks, xTimeNow, xMessage.u.xTimerParameters.xMessageValue ) != pdFALSE )
                    {
                        /* 在加入列表前已經超時，執行對應的回調函數 */
                        pxTimer->pxCallbackFunction( ( TimerHandle_t ) pxTimer );
                        traceTIMER_EXPIRED( pxTimer );

                        /*如果是週期定時器*/
                        if( pxTimer->uxAutoReload == ( UBaseType_t ) pdTRUE )
                        {
                            /* 發送消息，通知守護任務將定時器插入當前列表 */
                            xResult = xTimerGenericCommand( pxTimer, tmrCOMMAND_START_DONT_TRACE, xMessage.u.xTimerParameters.xMessageValue + pxTimer->xTimerPeriodInTicks, NULL, tmrNO_DELAY );
                            configASSERT( xResult );
                            ( void ) xResult;
                        }
                        else
                        {
                            mtCOVERAGE_TEST_MARKER();
                        }
                    }
                    else
                    {
                        mtCOVERAGE_TEST_MARKER();
                    }
                    break;

                /* 停止定時器 */
                case tmrCOMMAND_STOP :
                case tmrCOMMAND_STOP_FROM_ISR :
                    /* 定時器已經從活躍列表中移除，所以什麼都不做 */
                    break;

                /* 改變定時器週期 */
                case tmrCOMMAND_CHANGE_PERIOD :
                case tmrCOMMAND_CHANGE_PERIOD_FROM_ISR :
                    /* 取出新的頻率 */
                    pxTimer->xTimerPeriodInTicks = xMessage.u.xTimerParameters.xMessageValue;
                    configASSERT( ( pxTimer->xTimerPeriodInTicks > 0 ) );

                    /* 計算超時時間，超時時間沒過則加入活躍列表 */
                    ( void ) prvInsertTimerInActiveList( pxTimer, ( xTimeNow + pxTimer->xTimerPeriodInTicks ), xTimeNow, xTimeNow );
                    break;

                /* 刪除定時器 */
                case tmrCOMMAND_DELETE :
                    #if( ( configSUPPORT_DYNAMIC_ALLOCATION == 1 ) && ( configSUPPORT_STATIC_ALLOCATION == 0 ) )
                    {
                        /* 釋放軟件定時器內存 */
                        vPortFree( pxTimer );
                    }
                    #elif( ( configSUPPORT_DYNAMIC_ALLOCATION == 1 ) && ( configSUPPORT_STATIC_ALLOCATION == 1 ) )
                    {
                        if( pxTimer->ucStaticallyAllocated == ( uint8_t ) pdFALSE )
                        {
                            /* 釋放軟件定時器內存 */
                            vPortFree( pxTimer );
                        }
                        else
                        {
                            mtCOVERAGE_TEST_MARKER();
                        }
                    }
                    #endif /* configSUPPORT_DYNAMIC_ALLOCATION */
                    break;

                default :
                    /* Don't expect to get here. */
                    break;
            }
        }
    }
}

4 軟件定時器的使用

4.1 軟件定時器控制塊（結構體）

/* 軟件定時器結構體 */
typedef struct tmrTimerControl
{
    const char *pcTimerName;            /* 定時器名字 */
    ListItem_t xTimerListItem;          /* 定時器列表項 */
    TickType_t xTimerPeriodInTicks;     /* 定時器定時時間 */
    UBaseType_t uxAutoReload;           /* 定時器週期模式 */
    void *pvTimerID;                    /* 定時器ID */
    TimerCallbackFunction_t pxCallbackFunction; /* 定時器回調函數 */

    #if (configUSE_TRACE_FACILITY == 1)
        UBaseType_t uxTimerNumber;
    #endif

    #if ((configSUPPORT_STATIC_ALLOCATION == 1) && (configSUPPORT_DYNAMIC_ALLOCATION == 1))
        uint8_t ucStaticallyAllocated; /*標記定時器使用的內存， 刪除時判斷是否需要釋放內存*/
    #endif
}xTIMER;
typedef xTIMER Timer_t;

4.2 創建一個軟件定時器

#if( configSUPPORT_DYNAMIC_ALLOCATION == 1 )
TimerHandle_t xTimerCreate( const char * const pcTimerName,        /* 定時器名字 */
                           const TickType_t xTimerPeriodInTicks,   /* 定時器定時時間 */
                           const UBaseType_t uxAutoReload,         /* 定時器週期模式 */
                           void * const pvTimerID,                 /* 定時器ID */
                           TimerCallbackFunction_t pxCallbackFunction ) /* 定時器回調函數 */
{
    Timer_t *pxNewTimer;

    /*為軟件定時器申請內存*/
    pxNewTimer = ( Timer_t * ) pvPortMalloc( sizeof( Timer_t ) );

    if( pxNewTimer != NULL )
    {
        prvInitialiseNewTimer( pcTimerName, xTimerPeriodInTicks, uxAutoReload, pvTimerID, pxCallbackFunction, pxNewTimer );

        #if( configSUPPORT_STATIC_ALLOCATION == 1 )
        {
            /* 定時器可以靜態創建，也可以動態創建，注意這個計時器是動態創建的，以防稍後刪除計時器 */
            pxNewTimer->ucStaticallyAllocated = pdFALSE;
        }
        #endif /* configSUPPORT_STATIC_ALLOCATION */
    }

    return pxNewTimer;
}
#endif /* configSUPPORT_STATIC_ALLOCATION */

成功申請定時器後， 定時器並沒有開始工作， 需要調用啟動或復位等API函數將該定時器中的 xTimerListItem 插入到定時器管理鏈表中， Daemon 任務才能在該定時器設定的溢出時刻調用其回調函數。

4.3 啟動定時器

當用戶創建並啟動一個軟件定時器時， FreeRTOS會根據當前系統時間及用戶設置的定時確定該定時器喚醒時間，並將該定時器控制塊掛入軟件定時器列表

下面來看一下當啟動多個軟件定時器時，軟件定時器列表是如何來管理這些定時器的：

例如：系統當前時間xTimeNow值為0，注意：xTimeNow其實是一個局部變量，是根據xTaskGetTickCount()函數獲取的，實際它的值就是全局變量xTickCount的值，表示當前系統時間。

4.3.1 例子1

  • 在當前系統中已經創建並啟動了1個定時時間為200定時器Timer1
  • 當系統時間xTimeNow為20的時候，用戶創建並且啟動一個定時時間為100的定時器Timer2，此時Timer2的溢出時間xTicksToWait就為定時時間+系統當前時間（100+20=120），然後將Timer2按xTicksToWait升序插入軟件定時器列表中
  • 當系統時間xTimeNow為40的時候，用戶創建並且啟動了一個定時時間為50的定時器Timer3，那麼此時Timer3的溢出時間xTicksToWait就為40+50=90，同樣安裝xTicksToWait的數值升序插入軟件定時器列表中

4.3.2 例子2

創建並且啟動在已有的兩個定時器中間的定時器也是一樣的:

  • 創建定Timer1並且啟動後，假如系統經過了50個tick， xTimeNow從0增長到50，與Timer1的xTicksToWait值相等， 這時會觸發與Timer1對應的回調函數，從而轉到回調函數中執行用戶代碼，同時將Timer1從軟件定時器列表刪除，如果軟件定時器是週期性的，那麼系統會根據Timer1下一次喚醒時間重新將Timer1添加到軟件定時器列表中，按照xTicksToWait的升序進行排列。
  • 同理，在xTimeNow=40的時候創建的Timer3，在經過130個tick後（此時系統時間xTimeNow是40，130個tick就是系統時間xTimeNow為170的時候），與Timer3定時器對應的回調函數會被觸發，接著將Timer3從軟件定時器列表中刪除，如果是週期性的定時器，還會按照xTicksToWait升序重新添加到軟件定時器列表中。


5 總結與注意事項

  • 編譯定時器相關代碼， 如需要使用定時器，需要先在 FreeRTOSConfig.h 中正確配置宏 configUSE_TIMERS為 1
  • 軟件定時器使用了系統的一個隊列和一個任務資源，軟件定時器任務的優先級默認為configTIMER_TASK_PRIORITY， 如果優先級太低， 可能導致定時器無法及時執行，所以為了更好響應，該優先級應設置為所有任務中最高的優先級。
  • 定時器任務的消息隊列深度為configTIMER_QUEUE_LENGTH， 設置定時器都是通過發送消息到該隊列實現的
  • 定時器任務的堆棧大小默認為configTIMER_TASK_STACK_DEPTH個字節。
  • 軟件定時器的回調函數中應快進快出，絕對不允許使用任何可能引軟件定時器起任務掛起或者阻塞的API接口，在回調函數中也絕對不允許出現死循環。
  • 創建單次軟件定時器，該定時器超時執行完回調函數後，系統會自動刪除該軟件定時器，並回收資源。
