引入了Pygments解析器，這大大增加了GNU GLOBAL支持的語言

 2021-02-13 

從GNU GLOBAL 6.3.2開始，包括使用Pygments的插件解析器(以下稱為Pygments解析器)。
它利用Pygments(一種用Python編寫的語法突出顯示器)來標記Pygments支持的多種類型的編程語言。

具體來說，GLOBAL-6.3.2(2014年9月4日發布)的新功能包括25種類型，其中包括只能搜索符號標簽的類型(如GLOBAL的作者所述)，它應支持約150種語言。
我個人使用它來標記Ruby，Go，JavaScript等。

盡管它是這樣的Pygments解析器，但似乎很難理解如何安裝它，因為GLOBAL插件解析器本身的設置有點困難，並且Pygments解析器取決於某些軟件。因此，我想解釋一下如何安裝Pygments解析器。

使用旺盛的標簽

在安裝之前需要一些先驗知識...

由

gtags生成的標簽包括定義標簽和符號標簽。參考標記有些特殊，並且沒有專用的索引標記。 1

僅Pygments只能生成符號標簽。您可以僅使用符號標簽來跳轉標簽，但是在這種情況下，所有定義和引用都將被視為符號。

能夠區分

定義標簽和參考標簽很方便。因此，如果安裝了Exuberant Ctags，則Pygments解析器將使用Exuberant Ctags生成定義標簽。
Pygments和ctags輸出的結合使各種定義參考符號的標記跳轉都可用。

請注意，僅針對Exuberant Ctags不支持的語言生成符號標簽。

安裝

對於Mac

Homebrew在Mac上易於使用。

1
$ brew install global --with-exuberant-ctags --with-pygments

這還將自動安裝Python Pygments軟件包和Exuberant Ctags。

對於類Unix OS

您需要什麼

  • Python 2.6或更高版本(使用GLOBAL-6.3.3或更高版本的Pygments解析器為3x即可)

  • Pygments Python軟件包(推薦1.6或更高版本)

  • 旺盛的Ctags 5.5或更高版本

安裝

安裝Pygments軟件包。

1
$ sudo pip install Pygments

安裝旺盛的Ctags。

我認為大多數發行版都有軟件包。
對於Ubuntu Linux：

1
$ sudo apt-get install exuberant-ctags

安裝GNU GLOBAL。

$ cd global-x.x.x
$ ./configure
$ make
$ sudo make install

嘗試移動

關於標簽

默認情況下，內置解析器在安裝GNU GLOBAL的情況下工作。
內置解析器僅支持有限的幾種語言，例如C / C，Java和PHP。

要使用

插件解析器，您需要在執行gtags時指定"標簽"，並指定要使用的解析器。

1
2
3
4
5
$ gtags --gtagslabel=pygments

# 環境変數で指定することもできます
$ export GTAGSLABEL=pygments
$ gtags

可以將/usr/local/etc/gtags/gtags.conf 2中定義的以下項指定為

標簽。

1
2

default:\
        :tc=native:
native:\
        :tc=gtags:tc=htags:
user:\
        :tc=user-custom:tc=htags:
ctags:\
        :tc=exuberant-ctags:tc=htags:
pygments:\
        :tc=pygments-parser:tc=htags:

如果未指定任何內容，則指定default。
使用Pygments解析器時，在標簽中指定pygments。
請注意，此文件中的tc=表示要即時讀取其他定義。

尖端

與內置拾音器配合使用

Pygments解析器很慢，因此建議您設置內置解析器以處理內置解析器支持的文件，而Pygments解析器則處理其餘文件。
同時，它確保Pygments解析器在不設置環境變量GTAGSLABEL和--gtagslabel選項的情況下工作。

復制名稱為~/.globalrc的

配置文件。

1
$ cp /usr/local/etc/gtags/gtags.conf ~/.globalrc    # gtags.conf の場所は環境により読み替えてください

編輯

~/.globalrc。

1
2
3
4
5
default:\
        :tc=native:
↓
default:\
        :tc=native:tc=pygments:

這允許Pygments解析器在不設置環境變量GTAGSLABEL或--gtagslabel選項的情況下工作。
此外，內置解析器將優先用於內置解析器可以處理的文件。
